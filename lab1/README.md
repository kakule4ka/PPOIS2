# Система моделирования садового участка (Лабораторная работа №1)

Данный проект представляет собой программную систему для управления процессами садоводства и облагораживания участка, разработанную в рамках курса «Проектирование программного обеспечения интеллектуальных систем».

## Предметная область
Система моделирует работу садовода, включая управление жизненным циклом растений (от посадки до сбора урожая), контроль состояния ресурсов (почва, вода, инструменты) и поэтапное строительство инфраструктуры (зона отдыха).

## Основные сущности
В соответствии с требованиями варианта задания реализованы следующие сущности:
* **Участок**: Центральный узел управления данными и логикой.
* **Растения**: Объекты с конечным автоматом состояний (`PlantState`), подверженные влиянию ресурсов и сорняков.
* **Инфраструктура**: Зона отдыха с поэтапной застройкой и Система полива с ограниченной вместимостью.
* **Ресурсы**: Почва (влажность, удобрения) и Инструменты (лопата, грабли, молоток) с механикой износа.

## Реализованные операции
1. **Посадка и уход**: Посев семян, прополка сорняков и запуск цикла роста растений.
2. **Полив и удобрение**: Управление запасами воды (расход зависит от количества посадок) и обогащение почвы питательными веществами.
3. **Обслуживание инвентаря**: Контроль прочности и ремонт изношенных инструментов.
4. **Строительство**: Поэтапная расчистка и застройка зоны отдыха на участке.
5. **Декорирование**: Размещение лимитированного количества украшений после завершения строительства зоны отдыха.

## Технические особенности
* **PEP8 & Типизация**: Весь код написан с соблюдением стандартов оформления и содержит аннотации типов.
* **Обработка исключений**: Реализована иерархия собственных исключений (`GardenLogicError`) для предотвращения логических ошибок.
* **CLI интерфейс**: Взаимодействие с системой осуществляется через текстовое меню в реальном времени.
* **Unit-тестирование**: Проект покрыт тестами с использованием встроенной библиотеки `unittest`.

## Структура проекта
Система организована по модульному принципу для обеспечения чистоты кода и разделения ответственности:
* **`garden/entities.py`**: Описание базовых сущностей. Реализует логику конечных автоматов для растений и механики износа инструментов.
* **`garden/garden_plot.py`**: Главный контроллер. Связывает все компоненты в единый рабочий процесс (Фасад).
* **`garden/storage.py`**: Логика сохранения и загрузки состояния участка с использованием бинарной сериализации.
* **`garden/exceptions.py`**: Доменные ошибки. Содержит иерархию исключений для обработки некорректных ситуаций в логике.
* **`main.py`**: Точка входа в программу и реализация интерфейса командной строки.
* **`tests/`**: Пакет с модульными тестами для проверки бизнес-логики.

## Использование системы (CLI)
После запуска программы (`python main.py`) управление осуществляется через интерактивное меню:
* **Пункты 1, 4, 5, 6**: Управление жизненным циклом растений и состоянием почвы.
* **Пункты 2, 3**: Взаимодействие с системой полива (расход и пополнение воды).
* **Пункты 7, 8**: Строительство зоны отдыха (по 25% за этап) и размещение декораций.
* **Пункт 9**: Восстановление прочности используемых инструментов до 100%.
* **Пункт 0**: Завершение работы с сохранением текущего состояния участка в файл.
